## 代码简化原则（code-simplifier）

每次写代码或重构时，必须遵循以下优先级规则：

### 最高优先级（CRITICAL）
1. **上下文先行**：先读懂项目规范和现有代码风格，再动手改
2. **行为不变**：只改写法，不改功能（返回值、错误处理、副作用必须保持一致）

### 高优先级（HIGH）
3. **范围控制**：只改最近动过的代码，diff 要小，不做无关重构
4. **控制流简化**：
   - 用提前 return 减少嵌套（最多 2-3 层）
   - 用 guard clause 处理前置条件
   - 不用嵌套三元表达式
   - 用可选链（?.）和空值合并（??）

### 中高优先级（MEDIUM-HIGH）
5. **命名清晰**：变量名要有意义，避免缩写，数据用名词、动作用动词

### 中优先级（MEDIUM）
6. **去重但不过度抽象**：重复 3 次以上才提取，优先用数据驱动代替重复条件
7. **清理死代码**：删掉没用的代码和废话注释，保留解释"为什么"的注释

### 中低优先级（LOW-MEDIUM）
8. **TypeScript 惯用法**：用 strict types，不用 any；用 const assertions 和 readonly
